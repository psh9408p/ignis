# Cafe24 상품 상세 커스텀 옵션 UI

Cafe24 상품 상세페이지의 기본 옵션 선택 영역을 골라담기 형태의 커스텀 옵션 UI로 확장한 과제입니다.  
기본 Cafe24 옵션 선택, 선택상품 목록, 수량 변경, 총금액 계산, 장바구니/바로구매 흐름은 유지하면서 사용자에게 더 명확한 옵션 선택 경험을 제공합니다.

## 구현 요약

### 1. 커스텀 옵션 UI

- 상품 상세페이지에 `#cafe24-option-picker` 영역을 생성합니다.
- Cafe24 기본 텍스트 버튼 옵션은 화면에서 숨기되, 실제 선택 이벤트는 그대로 사용합니다.
- 옵션은 `10개입`, `30개입`, `50개입`, `100개입` 카드 형태로 노출됩니다.

### 2. Cafe24 기본 옵션과 동기화

- 커스텀 옵션 클릭 시 숨겨진 Cafe24 기본 옵션 버튼을 대신 클릭합니다.
- 선택 결과는 Cafe24의 `#totalProducts` 선택상품 목록에 생성됩니다.
- 수량 변경, 삭제, 총금액 계산, 장바구니 담기, 바로구매는 Cafe24 기본 구매 흐름을 사용합니다.

### 3. 중복 방지 및 추가 가능 횟수

- 옵션값 suffix를 기준으로 같은 개입수의 추가 가능 횟수를 제한합니다.
- 예: `30개입_1`, `30개입_2`가 있으면 `30개입`은 최대 2회 선택 가능합니다.
- 현재 설정은 Cafe24 옵션 구성에 맞춰 모든 개입수의 `maxSelectable`을 `2`로 맞췄습니다.
- 실제 Cafe24 옵션 수보다 크게 설정해도 없는 옵션은 추가되지 않도록 `picker-ui.js`에서 안전 제한을 적용합니다.

### 4. 외부 설정 JS

변경 가능한 표시 데이터는 `js/picker-config.js`에서 관리합니다.

```js
{
  packName: '30개입',
  displayName: '30개입',
  maxSelectable: 2,
  badge: '인기',
  description: '가장 많이 선택하는 구성입니다.',
  priceText: '70,500원',
  discountText: '29% 할인',
  unitPriceText: '1개 2,350원',
  recommended: true,
}
```

설정 항목:

- `packName`: Cafe24 옵션명에서 suffix를 제외한 기준명입니다.
- `displayName`: 화면에 표시할 옵션명입니다.
- `maxSelectable`: 해당 개입수의 최대 추가 가능 횟수입니다.
- `badge`: 추천/강조 뱃지 문구입니다.
- `description`: 옵션 설명 문구입니다.
- `priceText`: 옵션 가격 표시 문구입니다.
- `discountText`: 할인율 표시 문구입니다.
- `unitPriceText`: 개당 가격 표시 문구입니다.
- `recommended`: 추천 옵션 여부입니다.

### 5. 선택 상태 표시

- 선택된 옵션은 배경색과 라디오 표시로 활성 상태가 보입니다.
- 최대 선택 횟수에 도달하면 해당 옵션은 비활성 상태가 됩니다.
- 선택 카운트는 `현재 선택 수 / 최대 선택 수` 형식으로 표시됩니다.

### 6. 반응형 대응

- `css/picker.css`에 모바일 브레이크포인트가 적용되어 있습니다.
- 작은 화면에서는 옵션 카드 간격, 라디오 위치, 이미지/수량 영역 크기가 축소됩니다.
- 선택상품 목록과 하단 구매 액션도 모바일에서 열 폭에 맞게 재배치됩니다.

## 구현 설명

### 기본 구조

커스텀 옵션 UI는 Cafe24 기본 옵션 기능을 대체하지 않고, 기존 옵션 선택 버튼 위에 별도의 표시 레이어를 추가하는 방식으로 구현했습니다.

- `product/detail.html`에서 커스텀 옵션용 CSS/JS 파일을 로드합니다.
- `js/picker-ui.js`가 Cafe24 기본 옵션 DOM을 읽어 옵션 목록을 분석합니다.
- 분석한 옵션값을 기준으로 사용자에게 보이는 카드형 옵션 UI를 생성합니다.
- 사용자가 커스텀 옵션을 클릭하면 실제로는 숨겨진 Cafe24 기본 옵션 버튼이 클릭됩니다.
- 이후 선택상품 목록, 수량 변경, 총금액 계산, 장바구니, 바로구매는 Cafe24 기본 로직을 그대로 사용합니다.

### 옵션값 처리 방식

Cafe24 옵션값은 `30개입_1`, `30개입_2`처럼 suffix가 붙은 형태를 기준으로 처리합니다.

- 화면 표시용 옵션명은 suffix를 제거한 `30개입`을 사용합니다.
- 같은 개입수의 suffix 개수를 기준으로 추가 가능 횟수를 판단합니다.
- 예를 들어 `30개입_1`, `30개입_2`가 존재하면 `30개입`은 최대 2회까지 추가 가능합니다.
- 현재 Cafe24 실제 옵션 구성에 맞춰 `10개입`, `30개입`, `50개입`, `100개입` 모두 최대 2회 선택 가능하도록 설정했습니다.

### 외부 설정 데이터

옵션별 표시 정보는 `js/picker-config.js`로 분리했습니다. 운영자가 문구나 표시값을 바꿔야 할 때 UI 로직 파일을 수정하지 않고 설정 파일만 수정할 수 있습니다.

- 뱃지 문구
- 옵션 설명
- 추천 옵션 여부
- 가격 표시 문구
- 할인 표시 문구
- 개당 가격 표시 문구
- 최대 추가 가능 횟수

단, 실제 구매 금액과 선택상품 생성은 Cafe24 기본 옵션 데이터를 기준으로 동작합니다. 외부 설정 JS의 가격 문구는 사용자에게 보여주는 표시 데이터이며, 결제 로직 자체를 임의로 계산하지 않습니다.

### 상태 동기화 방식

커스텀 UI와 Cafe24 기본 선택상품 목록이 어긋나지 않도록 DOM 상태를 계속 동기화합니다.

- 옵션 선택 시 Cafe24 기본 옵션 버튼을 트리거합니다.
- `#totalProducts` 영역의 선택상품 목록을 읽어 현재 선택 상태를 갱신합니다.
- 이미 선택된 suffix 옵션은 다시 선택하지 않고, 다음 선택 가능한 suffix 옵션을 선택합니다.
- 최대 선택 횟수에 도달한 옵션은 비활성화하여 추가 클릭을 막습니다.
- 선택 상태, 선택 개수, 비활성 상태를 카드 UI에 반영합니다.

### 스타일 구현

CSS는 기존 상품 상세 레이아웃을 유지하면서 커스텀 옵션 영역만 별도로 스타일링했습니다.

- 선택 전/선택 후/비활성 상태를 구분합니다.
- 옵션 선택 라디오 표시가 중앙에 맞도록 정렬했습니다.
- 선택상품 목록에는 불필요한 배경색이 들어가지 않도록 정리했습니다.
- PC와 모바일에서 카드 간격, 폰트 크기, 구매 액션 영역이 깨지지 않도록 반응형 스타일을 적용했습니다.

### QA 중 확인된 보완 사항

- Cafe24 실제 옵션 구성과 외부 설정의 `maxSelectable` 값이 다르면 UI 제한 횟수가 달라질 수 있어, 현재는 실제 옵션 구성에 맞춰 모든 개입수를 2회로 설정했습니다.
- 가격 표시 문구는 `picker-config.js`에서 관리하지만, 실제 결제 금액은 Cafe24 선택상품 금액을 기준으로 합니다.
- Cafe24 스킨 캐시나 배포 반영 시점에 따라 운영 페이지에서 최신 JS/CSS 반영까지 시간이 걸릴 수 있습니다.

## 확인한 주요 동작

- `30개입`을 2회 선택하면 `30개입_1`, `30개입_2`가 선택상품 목록에 생성됩니다.
- 2회 선택 후 `30개입` 카드는 `2 / 2 선택`으로 표시되고 비활성화됩니다.
- 선택상품 수량을 증가시키면 상품 금액과 총 상품금액이 함께 갱신됩니다.
- 장바구니 담기 클릭 시 Cafe24 장바구니 페이지로 이동하며 담긴 수량이 반영됩니다.
- 바로 구매 클릭 시 로그인/주문서 흐름으로 진입합니다.

## 제출 메모

이 작업은 Cafe24 기본 구매 로직을 재구현하지 않고, 기본 옵션 시스템 위에 커스텀 UI를 얹는 방식으로 구현했습니다.  
따라서 실제 결제 금액, 선택상품 생성, 장바구니/바로구매 흐름은 Cafe24 기본 기능을 따르며, UI 표시와 선택 편의성만 커스텀 레이어에서 담당합니다.
